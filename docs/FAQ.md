# FAQ

## Is TorX One a normal chat app?

No. TorX One is a privacy-first decentralized messenger prototype. It is designed around local identity, Tor hidden services, and nearby transport instead of a central chat server.

## Does TorX One need a phone number?

No. Identity is based on cryptographic keys, not a phone number.

## What does Tor do in the app?

Tor helps the device become reachable through an onion address without exposing the user's normal public IP address.

## Is Tor the same as end-to-end encryption?

No. Tor protects the route. Encryption protects the message content. TorX One uses both ideas as separate layers.

## Can someone become Founder by changing their name?

No. Founder status is tied to the official Founder signing public key, not the display name.

## Why does the QR contact string still start with `astra:`?

That is the current legacy contact URI prefix from the earlier project name. It remains in the protocol for compatibility until the app code migrates to a new prefix.

## Should testers use the GitHub repo link or the release link?

Use the release link:

https://github.com/Nikhilesh-CS/TorX-One/releases/latest

That page is cleaner for testers because it points directly to the APK.

## Is TorX One production ready?

Not yet. It is beta software and should be treated as a prototype until independent security review and further hardening are complete.
